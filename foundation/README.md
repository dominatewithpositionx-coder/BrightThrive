# BrytThrive Foundation Documents

This directory contains the canonical foundation documents that govern BrytThrive's product philosophy, constitutional principles, and operating practices.

These documents are ratified records — they define how the product thinks, not just what it does. Product decisions, design choices, and engineering work that conflict with a ratified principle must be redesigned before shipping.

## How to Amend the Constitution

1. Propose the new principle or amendment in writing.
2. Reference which existing articles it extends, modifies, or supersedes.
3. Record the amendment in the Constitution's [Amendment History](./CONSTITUTION.md#amendment-history) with date and status.
4. Update `PRODUCT_OPERATING_SYSTEM.md` if new permanent product tests are introduced.
5. Update this README's version record.
6. Commit as `docs: ratify <Amendment Name>`.
