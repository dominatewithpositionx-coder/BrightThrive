- [ ] `add_coins` RPC exists in Supabase — run the verification query below
- [ ] Do a full end-to-end smoke test yourself before sending invites

### Verify `add_coins` RPC in Supabase SQL Editor
```sql
-- Check that the RPC exists
SELECT routine_name, routine_type
FROM information_schema.routines
WHERE routine_schema = 'public'
  AND routine_name = 'add_coins';

-- Check its parameter signature
SELECT
  p.parameter_name,
  p.data_type,
  p.parameter_mode
FROM information_schema.parameters p
JOIN information_schema.routines r ON r.specific_name = p.specific_name
WHERE r.routine_schema = 'public'
  AND r.routine_name = 'add_coins'
ORDER BY p.ordinal_position;
```

**Expected parameters:**
| Name | Type | Mode |
|------|------|------|
| p_child_id | uuid | IN |
| p_amount | integer | IN |
| p_type | text | IN |
| p_description | text | IN |
| p_mission_id | uuid | IN (nullable) |
| p_reward_id | uuid | IN (nullable) |

If the RPC is missing, the mission completion and reward redemption flows will silently fail on coin updates (missions still mark complete, but coins won't change).

## Test Scenarios

### 1. Parent Onboarding

**Goal:** Parent signs up, receives welcome email, reaches dashboard.

Steps:
1. Go to `/onboarding`
2. Complete all wizard steps (mood, schedule, location optional)
3. Enter email + password → click "Create account"
4. **If email confirmation is required:** check inbox, click confirm link, then log in
5. **If no confirmation required:** should land on `/dashboard` directly

Expected:
- Welcome email from Wayne arrives within 2 minutes
- Dashboard shows "Add your first child" empty state
- No errors in browser console

---

### 2. Add a Child

**Goal:** Parent successfully adds a child profile.

Steps:
1. Dashboard → Children → "Add Child"
2. Enter name and age (screen limit optional)
3. Submit

Expected:
- Child card appears immediately
- Dashboard home shows child card with 0 coins

---

### 3. Child Mood Flow

**Goal:** Child selects mood and sees an appropriate response.

Steps:
1. Go to `/child`
2. Select the child's name (enter PIN if set)
3. Tap a mood (e.g. "Happy", "Tired", "Frustrated")
4. Read the mood response screen
5. Tap "Let's see my missions!"

Expected:
- Mood response matches the selected mood
- Transitions are smooth with `animate-fade-in`
- No loading spinner stays stuck

---

### 4. Mission Generation

**Goal:** 5 missions are generated by Claude Haiku.

Steps:
1. After mood flow, missions screen loads
2. If no missions exist, they auto-generate
3. If missions exist, tap "New missions" button

Expected:
- 5 missions appear within ~5 seconds
- Missions are age-appropriate and varied
- If generation fails, 5 fallback missions appear (not an error screen)
- Rapid re-taps show "Just a moment!" message (rate limit working)

**If location is set in Settings:**
- Missions may reference weather (e.g. "It's rainy today — read a book inside")

---

### 5. Complete a Mission

**Goal:** Child completes a mission and earns coins.

Steps:
1. Tap a mission checkbox
2. Observe confetti
3. Check coin balance at top of screen
4. Tap same checkbox again to undo

Expected:
- Confetti fires on completion
- Coin balance increases by 10
- Undoing decreases coin balance by 10
- Ledger entry visible in `/dashboard/history`

---

### 6. Create a Reward

**Goal:** Parent creates a reward that a child can work toward.

Steps:
1. Dashboard → Rewards → form at top
2. Enter reward name (e.g. "30 min gaming") and coin cost (e.g. 50)
3. Submit

Expected:
- Reward appears in list immediately
- Per-child buttons show affordability (green = can redeem, gray = cannot)

---

### 7. Reward Redemption

**Goal:** Parent redeems a reward for a child who can afford it.

Steps:
1. Ensure child has enough coins (complete missions until balance ≥ reward cost)
2. Dashboard → Rewards
3. Tap the child's button on the reward
4. Confirm in the modal

Expected:
- Confirmation modal shows child name, reward, and post-redemption balance
- Coins deduct correctly
- Redemption appears in history
- If "Reward Redemption Alerts" is enabled in Settings → parent receives email

## Known Limitations for Pilot

1. **Weekly summary emails** — template exists but send is not automated. Tell pilot families this is coming soon.
2. **Password change** — not available in settings. Families must use "Forgot Password" flow to change password.
3. **Delete account** — not available in app. Contact Wayne directly.
4. **Multiple devices** — data syncs via Supabase real-time. If the same child is open on two devices simultaneously, coin updates may race. Advise one device at a time.
5. **Rate limiting** — mission generation is limited to once per 60 seconds per child. This is intentional.
6. **Email sender domain** — emails come from `notifications@resend.dev`. Some clients may flag this. Tell families to check spam and whitelist.
